# Why Your Message Appears Twice

The issue is that your **server is likely echoing the message back to you**. When you send a message, it appears twice because:

1. **You send the message** → Server receives it
2. **Server broadcasts to ALL clients** (including you) → You receive your own message back
3. Result: You see it twice

This is standard chat server behavior. The question is whether you want to:

## Option A: Prevent the echo (server-side)

Modify your server to skip sending the message back to the sender:

```javascript
// When broadcasting, exclude the sender:
wss.clients.forEach((client) => {
  if (client !== senderSocket && client.readyState === WebSocket.OPEN) {
    client.send(message); // Don't send back to sender
  }
});
```

## Option B: Filter duplicates (client-side)

In your `onmessage` handler, check if the message is from yourself:

```javascript
socket.onmessage = (event) => {
  try {
    const data = JSON.parse(event.data);
    const time = new Date(data.timestamp || Date.now()).toLocaleTimeString();
    const myName = nameInput.value.trim();

    // Only display if it's not your own message, OR always show if you want echoes
    if (data.name !== myName) {
      if (data.type === 'message') {
        addMessage(`[${time}] ${data.name}: ${data.text}`);
      } else if (data.type === 'join') {
        addMessage(`[${time}] ${data.name} joined`);
      } else if (data.type === 'leave') {
        addMessage(`[${time}] ${data.name} left`);
      } else if (data.type === 'error') {
        addMessage(`[ERROR] ${data.message}`);
      }
    }
  } catch (e) {
    addMessage('Invalid message received');
  }
};
```

## Option C: Keep the echo but show it differently

```javascript
if (data.type === 'message') {
  const prefix = data.name === myName ? '(You)' : '';
  addMessage(`[${time}] ${prefix}${data.name}: ${data.text}`);
}
```

**Which approach is best?** It depends on your preference:
- **Option A** is cleaner if you don't want echoes at all
- **Option B/C** gives you more control in the client
