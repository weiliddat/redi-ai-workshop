# Adding `/nick` Command to Your Chat Client

You need to make two changes:

## 1. Detect and send the `/nick` command (in your form submit handler)

Currently your submit handler only sends regular messages. You need to check for `/nick` first:

```javascript
chatForm.addEventListener('submit', (e) => {
  e.preventDefault();

  if (!socket || socket.readyState !== WebSocket.OPEN) {
    return;
  }

  const text = messageInput.value.trim();
  if (!text) return;

  // Check for /nick command
  if (text.startsWith('/nick ')) {
    const newName = text.slice(6).trim(); // Extract everything after "/nick "
    if (newName) {
      socket.send(JSON.stringify({
        type: 'nick',
        name: newName
      }));
    }
    messageInput.value = '';
    return;
  }

  // Regular message handling (existing code)
  socket.send(JSON.stringify({
    type: 'message',
    text: text
  }));

  messageInput.value = '';
});
```

## 2. Handle nick change broadcasts (in your `socket.onmessage` handler)

Add a case for when the server broadcasts a nick change (including your own):

```javascript
socket.onmessage = (event) => {
  try {
    const data = JSON.parse(event.data);

    const time = new Date(data.timestamp || Date.now()).toLocaleTimeString();

    if (data.type === 'message') {
      addMessage(`[${time}] ${data.name}: ${data.text}`);
    } else if (data.type === 'join') {
      addMessage(`[${time}] ${data.name} joined`);
    } else if (data.type === 'leave') {
      addMessage(`[${time}] ${data.name} left`);
    } else if (data.type === 'nick') {
      addMessage(`[${time}] ${data.oldName} is now known as ${data.name}`);
      // Optionally update the name input if this was your nick change
      if (data.name === nameInput.value) {
        nameInput.value = data.name;
      }
    } else if (data.type === 'list') {
      // Handle /list response (see below)
    } else if (data.type === 'error') {
      addMessage(`[ERROR] ${data.message}`);
    }
  } catch (e) {
    addMessage('Invalid message received');
  }
};
```

## Summary of Changes

| Location | Change |
|----------|--------|
| `chatForm` submit handler | Check for `/nick ` prefix, extract new name, send `{"type": "nick", "name": "..."}` |
| `socket.onmessage` | Add case for `data.type === 'nick'` to display the name change |

The server handles uniqueness validation and broadcasts the change to all clients—so your client just needs to display the message when received.
