You’re already *very close*—your server is sending a `timestamp`, and your client is even parsing it. You just need to consistently format and display it.

Here’s the clean step-by-step:

---

### 1. Ensure every message has a timestamp
You already do this:

```js
const time = new Date(data.timestamp || Date.now()).toLocaleTimeString();
```

This means:
- If server sends `timestamp` → use it  
- Otherwise → fall back to current time

✅ Nothing to change here unless you want a custom format

---

### 2. Create a helper function (optional but cleaner)

Right now you're repeating timestamp logic inline. Extract it:

```js
function formatTime(ts) {
  return new Date(ts || Date.now()).toLocaleTimeString();
}
```

---

### 3. Use it for every message type

Update your `onmessage` handler:

```js
socket.onmessage = (event) => {
  try {
    const data = JSON.parse(event.data);
    const time = formatTime(data.timestamp);

    if (data.type === 'message') {
      addMessage(`[${time}] ${data.name}: ${data.text}`);
    } else if (data.type === 'join') {
      addMessage(`[${time}] ${data.name} joined`);
    } else if (data.type === 'leave') {
      addMessage(`[${time}] ${data.name} left`);
    } else if (data.type === 'nick') {
      addMessage(`[${time}] ${data.oldName} is now ${data.name}`);
    } else if (data.type === 'list') {
      addMessage(`[${time}] Users: ${data.names.join(', ')}`);
    } else if (data.type === 'error') {
      addMessage(`[${time}] ERROR: ${data.message}`);
    }
  } catch (e) {
    addMessage('Invalid message received');
  }
};
```

---

### 4. (Optional) Improve timestamp format

If you want something nicer than default locale:

**24-hour format:**
```js
return new Date(ts).toLocaleTimeString([], { hour: '2-digit', minute: '2-digit' });
```

**Include date:**
```js
return new Date(ts).toLocaleString();
```

### 5. (Optional) Style timestamps differently

function addMessage(html) {
