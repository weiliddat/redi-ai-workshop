A `SyntaxError: Unexpected token` when a message arrives almost always means your code is trying to **parse a message as JSON, but the message is not a valid JSON string.**

Here are the most common scenarios and how to fix them:

### 1. The Server is sending a raw string, not JSON
If your server sends the word `Hello` or `OK` instead of `{"status": "OK"}`, `JSON.parse()` will crash.

*   **The Error:** `JSON.parse("Hello")` → `Unexpected token H in JSON at position 0`
*   **The Fix:** Wrap your parsing in a `try...catch` block.

```javascript
socket.onmessage = function(event) {
    try {
        const data = JSON.parse(event.data);
        console.log("Parsed data:", data);
    } catch (e) {
        console.log("Received non-JSON message:", event.data);
    }
};
```

### 2. Double Encoding (The "Stringified String")
Sometimes a server sends a string that was stringified twice (e.g., `"{\"id\": 1}"`). When you parse it once, you get a string. If you try to access a property on it, it fails; if you try to parse it again in the wrong way, it throws an error.

### 3. Syntax Errors in the JSON itself
JSON is very strict. It will fail if:
*   **Single quotes** are used: `{'id': 1}` (must be `{"id": 1}`)
*   There is a **trailing comma**: `{"id": 1,}`
*   An unescaped character is present (like a newline or a tab).

**How to debug:**
Add `console.log("Raw data:", event.data);` immediately before your `JSON.parse()` line. Look closely at the output:
*   Does it look like an Object?
*   Is it empty?
*   Is it an error message from your server (like an HTML 500 error page)?

### 4. Receiving "Heartbeats" or "Pings"
Many WebSocket setups (like Socket.io or AWS API Gateway) send "Heartbeat" messages to keep the connection alive. These are often just a single character (like `1` or `a`).
*   **The Fix:** Check the length or content before parsing.
```javascript
if (event.data === "ping") return; 
const data = JSON.parse(event.data);
```

### 5. The "Unexpected Token <" (HTML Error)
If the error says `Unexpected token <`, it means your JavaScript received **HTML** instead of JSON. 
*   **Why?** This usually happens if your backend crashes or a session expires, and the server sends back an HTML Error page or a Login page instead of the data you expected.

### Summary Checklist:
1.  **Log the raw data** before parsing: `console.log('|' + event.data + '|')`.
2.  **Use `try...catch`** to prevent the script from dying.
3.  **Validate on the backend** that you are using `json_encode` (PHP), `JSON.stringify` (Node), or `json.dumps` (Python) before sending.
