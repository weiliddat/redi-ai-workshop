This error means your browser tried to establish a WebSocket connection to `ws://localhost:3000`, but the connection failed. Here are the most common causes and solutions:

## 🔍 **Most Likely Causes:**

### 1. **Server Not Running**
- Your WebSocket server isn't running on port 3000
- **Check:** Open terminal and run:
  ```bash
  netstat -an | grep 3000  # Linux/Mac
  netstat -an | find "3000"  # Windows
  ```
  Or simply visit `http://localhost:3000` in browser (if it's an HTTP endpoint too).

### 2. **Wrong Port/Path**
- The WebSocket server might be on a different port (e.g., 8080, 5000)
- Or on a different path: `ws://localhost:3000/ws` instead of just `ws://localhost:3000`
- **Check:** Your server configuration and client connection code.

### 3. **Protocol Mismatch**
- Using `ws://` when server expects `wss://` (HTTPS/WSS)
- Or vice versa: using `wss://` on a non-HTTPS server
- **Fix:** Match the protocol to your server setup.

### 4. **CORS/Origin Issues** (Less common for WS)
- Some servers reject connections from certain origins
- **Check:** Server logs for "Invalid origin" messages.
- **Fix:** Configure server to accept your origin (e.g., `http://localhost:3000`).

### 5. **Server Not Configured for WebSocket**
- Server might be HTTP-only and not handling WebSocket upgrade handshake
- **Check:** Server code should handle the `Upgrade: websocket` header.
- **Example issues:**
  - Express server without WebSocket middleware
  - API server running but not WS server

### 6. **Firewall/Antivirus Blocking**
- Local firewall or antivirus might block port 3000
- **Test:** Temporarily disable firewall/AV or try a different port.

## 🛠️ **Debug Steps:**

### **Step 1: Check Browser Network Tab**
1. Open DevTools (F12) → **Network tab**
2. Filter by "WS" (WebSocket)
3. Look at the failed connection details:
   - Status code
   - Response headers
   - Error message (hover over red entry)

### **Step 2: Test Connection Manually**
Use `wscat` (install via `npm install -g wscat`):
```bash
wscat -c ws://localhost:3000
```
If this fails too, it's definitely a server issue.

### **Step 3: Verify Server Code**
**Node.js example (ws library):**
```javascript
const WebSocket = require('ws');
const wss = new WebSocket.Server({ port: 3000 });

wss.on('connection', (ws) => {
  console.log('Client connected');
});
```
**Python example:**
```python
import asyncio, websockets
async def handler(websocket):
    print("Client connected")
start_server = websockets.serve(handler, "localhost", 3000)
```

### **Step 4: Check for Mixed Content**
- If your page is served via **HTTPS** (`https://`), you **must** use `wss://` (secure WebSocket)
- If page is `http://`, you can use `ws://`

### **Step 5: Look at Server Logs**
Check if the server shows:
- "Listening on port 3000"
- Any connection attempts
- Any errors during startup

## 📝 **Common Fixes:**

| Symptom | Likely Fix |
|---------|------------|
| `Connection refused` | Server not running on that port |
| `Handshake error` | Server not handling upgrade header |
| `Invalid URL` | Wrong path in client URL |
| Works in wscat but not browser | CORS/origin issue, browser extension blocking |

## 🔥 **Quick Checklist:**
1. ✅ Server process is running (`ps aux | grep node` or similar)
2. ✅ Server is bound to `localhost:3000` (not just `127.0.0.1` if accessing by name)
3. ✅ Client URL matches server exactly (protocol, host, port, path)
4. ✅ No browser extensions blocking WS (try incognito mode)
5. ✅ No proxy/VPN interfering
6. ✅ Server has WebSocket handler, not just HTTP routes
