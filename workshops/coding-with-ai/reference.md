# References

## How AI Assistance Impacts the Formation of Coding Skills

- **Source:** Anthropic Research (Jan 29, 2026)
- **Link:** https://www.anthropic.com/research/AI-assistance-coding-skills
- **Paper:** https://arxiv.org/abs/2601.20245

This study is useful for the workshop because it shows that AI use does not automatically produce learning.

Students learned less when they delegated the thinking to AI. They learned more when they stayed active: asking conceptual questions, requesting explanations, reading code, and writing parts themselves.

For people coding with AI, the lesson is practical:

- do not accept code you cannot explain
- ask the agent to teach while it helps
- read diffs before committing
- use review and tests to check your understanding

## Humans Don't Just Sample

- **Source:** joyemang33.github.io (2026)
- **Link:** https://joyemang33.github.io/blog/2026/humans-dont-just-sample/

This post is useful for framing long-horizon work with AI.

The workshop uses it to support a practical point: students should not treat AI work as one prompt and one answer. Strong project work depends on memory, feedback, reflection, and adaptation over time.

For people coding with AI, the lesson is:

- build project memory in docs and instructions
- create feedback loops instead of relying on one-shot generation
- reflect after getting stuck
- improve how you and the agent work together over time

## Anthropic Prompt Library

- **Source:** Anthropic
- **Link:** https://platform.claude.com/docs/en/resources/prompt-library/library

A collection of prompt examples for common tasks.

Useful as a reference for structuring requests, but the workshop should not teach prompting as the only skill. Prompting matters most when combined with version control, understanding, and verification.
