Short answer: no—you can’t send a plain JavaScript object over a WebSocket. It has to be turned into something the protocol understands, like a string or binary data.

WebSockets don’t know anything about JS objects. They only send:
- strings
- binary data (Blob, ArrayBuffer, etc.)

So when you do:
socket.send({ type: 'message', text: 'hi' });

that won’t work—it’ll either throw an error or send something meaningless like "[object Object]".

`JSON.stringify` converts your object into a string:
JSON.stringify({ type: 'message', text: 'hi' })
→ '{"type":"message","text":"hi"}'

That string can be sent over the wire, and then the server can reconstruct the object using `JSON.parse`.

Think of it like this:
- `JSON.stringify` = packing the object into a transferable format
- `JSON.parse` = unpacking it on the other side

If you really wanted, you *could* send plain text (like `"hello"`), but then you lose structure (no `type`, no fields, etc.). JSON gives you a clean, predictable message format.
