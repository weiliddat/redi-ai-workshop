The error message "WebSocket connection to 'ws://localhost:3000' failed" indicates that your frontend tried to open a WebSocket connection, but the backend server didn't respond properly or isn't reachable. This is common in development environments and usually means the server isn't running, isn't configured to handle WebSockets, or there's a mismatch in how the connection is being established.

### Common Causes and Troubleshooting Steps

#### 1. 🚀 Server Not Running
Check if your backend server is active and running on port 3000.

```bash
# Check if port 3000 is in use
lsof -i :3000        # macOS/Linux
netstat -ano | findstr :3000  # Windows
```

- Ensure the server process (e.g., Node.js with Socket.io, WebSocket, FastAPI, etc.) is running.
- Restart the server if needed.

#### 2. ❌ WebSocket Support Not Enabled
Not all backends use WebSockets by default.

- **Node.js + Express**:
  ```bash
  npm install socket.io ws
  ```
  Make sure the server initializes a WebSocket or Socket.IO server:
  ```javascript
  const http = require('http');
  const socketIO = require('socket.io');

  const server = http.createServer(app);
  const io = socketIO(server); // Enable WebSocket support
  server.listen(3000);
  ```

- **Python (FastAPI)**:
  ```bash
  pip install fastapi websockets uvicorn
  ```
  And initialize WebSocket in your code:
  ```python
  app.add_route("/ws", ws_handler, methods=["GET"])
  # OR use websockets package to handle connections
  ```

- **Spring Boot or other frameworks**: Ensure WebSocket handlers and configuration are properly set up.

#### 3. 🌐 Wrong Protocol or Port
Verify your frontend is connecting to the correct URL and protocol:

- ✅ `ws://localhost:3000` or `wss://localhost:3000`
- ❌ `http://localhost:3000` (HTTP, not WebSocket)
- ❌ `ws://yourdomain.com` if running locally

In some cases, especially in development with proxies like Vite or Webpack dev server, you may need to proxy WebSocket connections:

```javascript
// vite.config.js
export default {
  server: {
    proxy: {
      '/ws': {
        target: 'http://localhost:3000',
        ws: true
      }
    }
  }
}
```

#### 4. 🛑 CORS and Security Issues
If you're serving web content from a different origin (e.g., `http://localhost:3001` connecting to `http://localhost:3000`), the browser may block the connection due to CORS restrictions.

- Configure your backend to allow the frontend's origin.
- Use secure protocols (`wss://`) in production.

#### 5. 🔧 Browser Console Debugging
To get more detail on what the failure is, open the browser console and look at the full error message:

```javascript
// Add this in your frontend code to capture the error
const socket = new WebSocket('ws://localhost:3000');

socket.onerror = function(error) {
  console.error("WebSocket Error:", error);
};
socket.onopen = function() {
  console.log("Connected");
};
socket.onclose = function() {
  console.log("Connection closed");
};
```

#### 6. ✅ Quick Checklist

- ✅ Backend: Server running on port 3000?
- ✅ WebSocket: Protocol is `ws://`, not `http://`
- ✅ Config: Server configured to accept WebSocket connections?
- ✅ CORS: Frontend origin allowed on backend?
- ✅ Proxy: Dev server configured to forward WebSocket traffic?

### Summary
This error typically points to a broken or unconfigured backend WebSocket setup. Start by confirming the server is running and supports WebSockets, then validate that your frontend is connecting correctly (protocol, port, origin), and check for any CORS or proxy configuration. Use browser console logs to pinpoint exact failure causes if needed.
