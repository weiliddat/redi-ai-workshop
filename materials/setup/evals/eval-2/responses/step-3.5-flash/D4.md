The issue is that you're likely **receiving the same message twice from the server** because your server is broadcasting messages back to **all connected clients including the sender**, and you're also displaying the sender's message locally in some implementations. However, looking at your client code, you **only display messages when they arrive via WebSocket** - you don't locally echo sent messages.

The most common causes for duplicate messages are:

## 1. **Server-Side Issue (Most Likely)**
Your WebSocket server is probably sending **two messages** for each chat message:
- One broadcast to all other clients
- One echo back to the sender
- OR the server has a bug causing duplicate broadcasts

Check your server code for something like:
```javascript
// Example problematic server pattern:
io.on('connection', (socket) => {
  socket.on('message', (data) => {
    // BAD: Sending twice
    socket.emit('message', data);  // Echo to sender
    socket.broadcast.emit('message', data);  // To others
    // OR accidentally calling broadcast twice
  });
});
```

## 2. **Client-Side Issue (Less Likely but Possible)**
If you **are** locally echoing messages (not in your current code), you'd see duplicates. Your current code only displays WebSocket messages, so this isn't the issue.

## 3. **Multiple Connections (Check This First!)**
Are you accidentally connecting **twice**? Your code has a potential issue:
- When you click `Connect`, it creates a new WebSocket without properly cleaning up old ones
- If the connection is slow and you click `Connect` again, you'll have **two active sockets**

**Fix for multiple connections:**
