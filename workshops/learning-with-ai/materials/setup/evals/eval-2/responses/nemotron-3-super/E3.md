**Short answer:**  
`WebSocket.send()` only accepts a **string**, a `Blob`, or an `ArrayBuffer` (or a view of one). If you give it a plain JavaScript object, the browser will first convert that object to a primitive value using its internal `ToPrimitive` algorithm ‚Äì which, for objects, ends up calling `Object.prototype.toString()` and producing the literal string `"[object Object]"`. That is not the structured data you intended to transmit, and the receiver would have no way to reconstruct the original object.

**What `JSON.stringify` does**

```javascript
socket.send(JSON.stringify({
    type: 'message',
    text: text
}));
```

* `JSON.stringify` turns the object into a **UTF‚Äë8 text string** that looks like:

```
{"type":"message","text":"Hello, world!"}
```

* That string is a valid WebSocket *text* frame, which the peer can safely receive and then parse back into an object with `JSON.parse(event.data)`.

**Why you can‚Äôt just send the object**

| What you pass to `socket.send()` | What the browser actually sends over the wire |
|----------------------------------|-----------------------------------------------|
| `string`                         | The string itself (UTF‚Äë8 encoded)            |
| `Blob` / `ArrayBuffer`           | The raw binary bytes                         |
| **object** (e.g. `{type:'msg'}`) | Result of `obj.toString()` ‚Üí `"[object Object]"` (a useless string) |
| `number`, `boolean`, `null`      | Converted to their string representation (`"5"`, `"true"`, `"null"` ) ‚Äì still not the structured data you wanted |

If you relied on the implicit conversion, the receiver would see `"[object Object]"` and would have no way to know what `"type"` or `"text"` were supposed to be. The message would be useless.

**Alternatives (when you don‚Äôt want JSON)**

* **Binary frames** ‚Äì you could encode the object yourself (e.g., using MessagePack, protobuf, or a custom binary format) and send an `ArrayBuffer`.  
* **Plain text** ‚Äì for extremely simple protocols you could send a raw string like `"message:Hello"` and parse it on the server, but you lose the ability to nest data or include special characters without extra escaping.

**Bottom line:**  
`JSON.stringify` is the conventional, safe way to turn a JavaScript object into a format that WebSocket can transmit as a text frame and that the other end can reliably turn back into an object. Sending the object directly would not give you the data you expect.
