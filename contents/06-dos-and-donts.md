# Do's and Don'ts

## Do's

**Ask why, not just what.**
"Why does this work?" teaches you more than "What is the answer?" You can use AI for both — but the first one builds skills you keep.

**Break problems into steps.**
Solve one piece at a time. Each step is a chance to understand before moving forward. One big prompt that returns one big solution gives you nothing to hold onto.

**Explain code back to yourself before using it.**
Read every line of AI-generated code out loud or in your head. If you can't explain a line, ask about it before pasting it in.

**Use AI to unstick yourself, not to think for you.**
"I'm stuck here, give me a hint" is different from "just solve this for me." One moves you forward. The other moves the AI forward.

**Keep your hands on the code.**
Type things out. Modify the examples. Break them intentionally and fix them. Active contact with code is how understanding gets built.

---

## Don'ts

**Don't paste the whole task and accept the first output.**
This is the delegation pattern. You get working code, no understanding, and a harder problem next time.

**Don't debug by asking AI to fix your errors.**
Error messages are information. Read them. Try to understand what broke and why before handing it to AI. If you need help, ask "What does this error mean?" not "Fix this".

**Don't accept code that works but you can't explain.**
Working code you don't understand is a liability. It will break at the worst moment and you won't know where to start.

**Don't skip the struggle.**
Frustration and confusion aren't signs you're doing it wrong. They're signs you're at the edge of what you know — which is exactly where learning happens. Sit there a little longer before reaching for AI.

**Don't use AI to avoid learning the fundamentals.**
HTML structure, Python syntax, basic data manipulation — these are the building blocks. If AI is writing them all, you're not building a foundation. You're borrowing one.

---

## The one thing to remember

> **If you can't explain it, you haven't learned it.**

That's the test. Apply it every time.

---

## What to do next

You now have the approach. The only thing left is to use it.

Pick one thing you're working on in your track. The next time you open an AI tool, try the conceptual inquiry pattern — ask why things work, build the answer yourself.

Do it once, and compare how you feel at the end versus the cold start activity. The difference is the point.

## Time

- 10 min — teacher presents
- 6 min — Q&A and close
