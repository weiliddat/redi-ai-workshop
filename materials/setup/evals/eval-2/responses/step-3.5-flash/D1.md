This error means your browser **failed to establish a WebSocket connection** to the server at `ws://localhost:3000`. Here's what's likely happening and how to fix it:

## **Most Common Causes & Solutions**

### 1. **Server isn't running or listening on port 3000**
- ✅ **Check if your server is running**: Look in your terminal/IDE to see if your Node.js/Python/etc. server is actively running.
- ✅ **Verify port**: The server might be on a different port (like 8080, 5000, etc.). Check your server config.
- ✅ **Test with curl**:
  ```bash
  curl -i -N -H "Connection: Upgrade" -H "Upgrade: websocket" \
       -H "Host: localhost:3000" -H "Origin: http://localhost:3000" \
       http://localhost:3000
  ```
  If you get `HTTP/1.1 101 Switching Protocols`, WebSocket is working. If not, the server isn't configured for WebSocket.

### 2. **Server not configured for WebSocket upgrade**
WebSocket connections start as HTTP requests with special headers (`Upgrade: websocket`, `Connection: Upgrade`). If your server doesn't handle these, it will reject the connection.

**For Node.js with `ws` library:**
```javascript
const WebSocket = require('ws');
const wss = new WebSocket.Server({ port: 3000 });
// Must handle upgrade requests
wss.on('connection', ws => {
  console.log('Client connected');
});
```

**For Express + ws:**
```javascript
const express = require('express');
const http = require('http');
const WebSocket = require('ws');

const app = express();
const server = http.createServer(app);
const wss = new WebSocket.Server({ server });

server.listen(3000);
```

### 3. **CORS/preflight issues** (less common for WS but possible)
WebSocket doesn't use CORS like regular HTTP, but the `Origin` header is checked by some servers. Try connecting without origin restrictions temporarily to test.

### 4. **Path mismatch**
If your client connects to `ws://localhost:3000/socket` but your server listens on `/ws`, it will fail. Ensure paths match.

### 5. **Firewall/antivirus blocking**
Temporarily disable firewall/antivirus to test. Some security software blocks local WebSocket connections.

## **Debugging Steps**

1. **Check server logs**: Look for errors when the connection attempt happens.
2. **Network tab in DevTools**:
   - Open Chrome DevTools → Network tab
   - Filter by "WS" (WebSocket)
   - Check the failed request details (status code, response headers)
3. **Test with a simple WebSocket client**:
   ```javascript
   // In browser console:
   const test = new WebSocket('ws://localhost:3000');
   test.onerror = e => console.error('WS Error:', e);
   ```
4. **Verify server binding**:
   ```bash
   # Check what's listening on port 3000
   lsof -i :3000  # Mac/Linux
   netstat -ano | findstr :3000  # Windows
   ```

## **Quick Checklist**

| Issue | How to Verify | Fix |
|-------|---------------|-----|
| Server not running | No process listening on 3000 | Start server |
| Wrong port | `curl localhost:3000` fails | Use correct port |
| No WS handler | HTTP 404/500 on upgrade request | Add WebSocket server code |
| Path mismatch | Client: `/socket`, Server: `/ws` | Match paths |
| IPv6 vs IPv4 | `localhost` resolves to `::1` | Try `ws://127.0.0.1:3000` |

## **Common Framework-Specific Issues**

- **React/Vue frontend**: Ensure you're not trying to connect before the server is ready (add retry logic).
- **Docker**: If server is in Docker, use `host.docker.internal` instead of `localhost` from browser.
- **Proxies**: Development proxies (like in `vite.config.js` or `webpack-dev-server`) may need WebSocket proxy configuration.
