This error typically means you're trying to **parse invalid JSON** when a message arrives, usually from a WebSocket, AJAX response, or similar event source. Here are the most common causes:

## 1. **Server is sending malformed JSON**
Your server might be sending:
- A **JavaScript object literal** (not JSON) like `{name: "John"}` instead of `{"name": "John"}`
- **Single quotes** instead of double quotes
- **Trailing commas** in objects/arrays
- **Unescaped characters** in strings
- **Non-JSON data** (HTML, plain text, etc.)

**Example of invalid JSON:**
```javascript
// Wrong:
{ name: 'John', age: 30 }  // keys need quotes, use double quotes
{ "name": "John", }        // trailing comma
[1, 2, 3,]                // trailing comma

// Correct:
{ "name": "John", "age": 30 }
[1, 2, 3]
```

## 2. **Double JSON parsing**
You might be **parsing JSON twice** if:
- You're using a library that **already parses JSON** (like Socket.IO, some WebSocket wrappers)
- Your message handler does `JSON.parse()` on data that's **already an object**

```javascript
// Problem: data is already an object (from library)
socket.on('message', (data) => {
  // DON'T do this if data is already parsed:
  const parsed = JSON.parse(data); // Error: Unexpected token o (object)
});
```

## 3. **Mixed content types**
Server might be sending:
- Empty responses (`""`)
- `null` or `undefined`
- Plain text like `"Hello"` instead of `{"message":"Hello"}`
- HTML error pages instead of JSON

## 4. **Character encoding issues**
- UTF-8 BOM (Byte Order Mark) at start
- Non-printable characters
- Incorrect encoding

## **How to debug:**

### **Step 1: Log the raw data**
```javascript
socket.onmessage = (event) => {
  console.log('Raw data:', event.data);
  console.log('Type:', typeof event.data);
  console.log('Length:', event.data.length);
  
  try {
    const parsed = JSON.parse(event.data);
    console.log('Parsed:', parsed);
  } catch (e) {
    console.error('JSON Parse Error:', e.message);
    console.error('First 100 chars:', event.data.substring(0, 100));
  }
};
```

### **Step 2: Check your server code**
Look for:
- How you're **stringifying** data: `JSON.stringify(obj)` (not `obj.toString()`)
- Content-Type headers: should be `application/json`
- No manual string concatenation for JSON

**Node.js/Express example:**
```javascript
// CORRECT
res.json({ message: 'hello' }); // Express auto-stringifies
res.send(JSON.stringify(data)); // Manual but correct

// WRONG
res.send('{message: "hello"}'); // Invalid JSON
```

### **Step 3: Validate with external tools**
- Paste the raw response into [jsonlint.com](https://jsonlint.com)
- Check browser **Network tab** → inspect the response body

## **Common fixes:**

### **If using raw WebSockets:**
```javascript
// Make sure server sends valid JSON
// Server-side (Node.js):
ws.send(JSON.stringify({ type: 'message', data: 'hello' }));

// Client-side:
socket.onmessage = (event) => {
  try {
    const data = JSON.parse(event.data);
    // Use data
  } catch (err) {
    console.error('Invalid JSON from server:', event.data);
  }
};
```

### **If using Socket.IO:**
Socket.IO **automatically parses JSON**, so don't parse again:
```javascript
// CORRECT - data is already an object
socket.on('message', (data) => {
  console.log(data.type); // Access directly
});

// WRONG - will cause error
socket.on('message', (data) => {
  const parsed = JSON.parse(data); // ❌ data is already object!
});
```

### **If server sends non-JSON:**
Either:
1. **Fix the server** to send proper JSON
2. **Handle the non-JSON case:**
```javascript
socket.onmessage = (event) => {
  try {
    const data = JSON.parse(event.data);
    // Handle JSON
  } catch (e) {
    // Handle plain text/other formats
    console.log('Plain text message:', event.data);
  }
};
```
