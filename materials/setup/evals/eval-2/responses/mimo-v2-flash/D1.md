The error `WebSocket connection to 'ws://localhost:3000' failed` means your browser attempted to establish a WebSocket connection to a server running on your local machine (`localhost`) at port `3000`, but the connection failed. Here's a breakdown of causes and solutions:

---

### **Common Causes & Fixes**
1. **No WebSocket Server Running**  
   - **Cause**: No application is listening on `ws://localhost:3000`.  
   - **Fix**:  
     - Start your WebSocket server (e.g., a Node.js/Express server with `ws` or Socket.IO).  
     - Verify the server is running on port `3000` using tools like `netstat` (Windows) or `lsof -i :3000` (Linux/macOS).  

2. **Port Mismatch**  
   - **Cause**: Your server might use a different port (e.g., `8080`, `5000`, `4000`).  
   - **Fix**:  
     - Check your server code for the correct port.  
     - Update the client code to match the server’s port:  
       ```javascript
       // Example: Change to port 5000 if server uses 5000
       const socket = new WebSocket('ws://localhost:5000');
       ```

3. **HTTP Server Not Running**  
   - **Cause**: Many WebSocket servers require an HTTP server first (e.g., Socket.IO).  
   - **Fix**: Ensure your server code creates an HTTP server and attaches WebSocket logic.  
     ```javascript
     // Example: Node.js with Express and Socket.IO
     const express = require('express');
     const http = require('http');
     const { Server } = require('socket.io');
     
     const app = express();
     const server = http.createServer(app);
     const io = new Server(server); // WebSocket attached to HTTP server
     
     server.listen(3000); // Now listen on port 3000
     ```

4. **Firewall/Antivirus Blocking**  
   - **Cause**: Local firewalls may block `localhost` connections.  
   - **Fix**: Temporarily disable your firewall/antivirus to test (or add an exception for port `3000`).

5. **Browser Extensions or Proxy Settings**  
   - **Cause**: Extensions (e.g., ad blockers) or proxies can interfere.  
   - **Fix**:  
     - Test in incognito mode (disables extensions by default).  
     - Check proxy settings in your OS/browser.

6. **CORS Issues**  
   - **Cause**: If your client and server are on different origins (e.g., client on `http://localhost:8080`, server on `ws://localhost:3000`).  
   - **Fix**: Enable CORS in your server. For Socket.IO:  
     ```javascript
     const io = new Server(server, {
       cors: {
         origin: "http://localhost:8080", // Client origin
         methods: ["GET", "POST"]
       }
     });
     ```

7. **HTTPS vs. HTTP**  
   - **Cause**: If your client is served over HTTPS, browsers block insecure WebSocket (`ws://`).  
   - **Fix**: Use `wss://` (secure WebSocket) or serve the client locally via HTTP (e.g., `http://localhost:8080`).

### **Debugging Steps**
1. **Test Server Connectivity**:  
   - Use `curl` or a WebSocket testing tool (e.g., [websocket.org/echo.html](https://www.websocket.org/echo.html)):  
     ```bash
     # Install a WebSocket CLI tool (e.g., wscat)
     npm install -g wscat
     wscat -c ws://localhost:3000
     ```
   - If `wscat` fails, the server isn’t running correctly.

2. **Check Server Logs**:  
   - Look for errors when starting your server (e.g., "EADDRINUSE" if port is occupied).

3. **Verify Network Requests**:  
   - Open browser dev tools (F12) → **Network** tab → check WebSocket requests. Look for HTTP status codes (e.g., 404, 500) or upgrade failures.

4. **Client-Side Code**:  
   - Add error handling to your WebSocket client:  
     ```javascript
     const socket = new WebSocket('ws://localhost:3000');
     socket.onerror = (error) => {
       console.error('WebSocket error:', error);
     };
     ```

### **Example Server Setup (Node.js/Express)**
```javascript
// server.js
const express = require('express');
const http = require('http');
const { Server } = require('socket.io');

const app = express();
const server = http.createServer(app);
const io = new Server(server, {
  cors: { origin: "*" } // Adjust for production
});

io.on('connection', (socket) => {
  console.log('Client connected!');
  socket.on('disconnect', () => console.log('Client disconnected'));
});

server.listen(3000, () => {
  console.log('WebSocket server running on port 3000');
});
```
Run with:  
```bash
npm install express socket.io
node server.js
```

### **Advanced Fixes**
- **Port Conflicts**: Kill processes using port 3000:  
  ```bash
  # Linux/macOS
  sudo lsof -i :3000 | kill -9 <PID>
  
  # Windows (PowerShell)
  Get-Process -Id (Get-NetTCPConnection -LocalPort 3000).OwningProcess | Stop-Process
  ```
- **Docker/Containers**: Ensure ports are exposed (e.g., `-p 3000:3000` in `docker run`).
- **Proxy Servers**: If using a reverse proxy (e.g., Nginx), configure WebSocket support:  
  ```nginx
  location / {
    proxy_pass http://localhost:3000;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
  }
  ```
