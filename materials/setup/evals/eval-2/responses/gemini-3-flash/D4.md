The reason your messages appear twice is a common behavior in WebSocket design: **The server is likely broadcasting your message back to everyone, including you.**

In your current logic, your code does "The Double-Tap":
1. **The Server Broadcast:** When you send a message, the server receives it and broadcasts it to all connected clients. Since you are a connected client, you receive your own message back via the `socket.onmessage` event, which triggers `addMessage`.
2. **Local Prediction (The potential culprit):** While your current code *doesn't* manually call `addMessage` inside the `submit` handler, most duplicate issues occur because the server logic is sending the message back to the sender, and the client treats it as a "new" incoming message.

### How to fix this

You have two main ways to solve this, depending on how you want your app to feel:

#### Option 1: Filter on the Client (Recommended)
You can assign a unique ID to each client or add a check to see if the message came from "you." However, since your server is already sending your name back, the easiest way is to let the **Server** handle the "source of truth."

If you want to keep your current code but stop the duplication, you need to change your **Server Code**. Usually, in WebSockets (like `ws` or `Socket.io`), you have two choices for broadcasting:
*   **Broadcast to all:** Sends to every single connection.
*   **Broadcast to others:** Sends to everyone *except* the sender.

**If you use Node.js `ws`, change your server loop to this:**
```javascript
wss.clients.forEach((client) => {
  // Check if it is NOT the sender and the connection is open
  if (client !== ws && client.readyState === WebSocket.OPEN) {
    client.send(JSON.stringify(data));
  }

```

If you **want** the server to send the message back to you (to confirm it was actually received and timestamped by the server), keep your code as it is, but **ensure you are not manually adding the message to the screen before sending it.**
