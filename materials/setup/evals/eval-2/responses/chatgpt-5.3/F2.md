You don’t need to touch the connection logic—this is all about intercepting user input before you send it and handling a new incoming event type.

Here’s what to change:

---

**1. Detect `/nick` in the submit handler**

Right now you always send messages as `{ type: "message" }`.  
You need to branch when the input starts with `/nick`.

Replace your `chatForm.addEventListener('submit', ...)` body with something like:

```js
chatForm.addEventListener('submit', (e) => {
  e.preventDefault();

  if (!socket || socket.readyState !== WebSocket.OPEN) {
    return;
  }

  const text = messageInput.value.trim();
  if (!text) return;

  // Handle /nick command
  if (text.startsWith('/nick ')) {
    const newName = text.slice(6).trim();

    if (!newName) {
      addMessage('[ERROR] Usage: /nick new-name');
      return;
    }

    socket.send(JSON.stringify({
      type: 'nick',
      name: newName
    }));
  } else if (text === '/list') {
    socket.send(JSON.stringify({ type: 'list' }));
  } else {
    socket.send(JSON.stringify({
      type: 'message',
      text: text
    }));
  }

  messageInput.value = '';
});
```

---

**2. Handle incoming `"nick"` events**

```js
else if (data.type === 'nick') {
```

**3. (Optional but useful) Track your own name locally**

Right now your UI doesn’t remember your name after connecting. If you want to reflect name changes locally (e.g., for display later), store it:

Inside `connect()`:

```js
if (data.oldName === currentName) {
```

Your server already does the heavy lifting.
