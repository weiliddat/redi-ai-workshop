# Removing the `onmessage` Handler

If you removed the `onmessage` handler, **you would lose all incoming communication** but the connection itself would still work.

## What would break:

1. **No incoming messages displayed** — Other users' messages would never appear in the chat
2. **No join/leave notifications** — You wouldn't see when users enter or exit the chat
3. **No error feedback** — Server-sent error messages would be silently ignored
4. **No echo of your own messages** — If your server echoes messages back, you wouldn't see them

## What would still work:

- ✅ Connecting to the server
- ✅ Disconnecting
- ✅ Sending messages (`socket.send()` still executes)
- ✅ UI updates based on connection state

## Practical impact:

The chat would feel broken from a user's perspective. They could connect, type, and hit send, but would see **absolutely no feedback** — making it appear as though messages aren't being sent at all, even though they technically are.

```javascript
// This still runs without onmessage:
socket.send(JSON.stringify({
  type: 'message',
  text: text
}));
// But you'd never know if it succeeded or if anyone received it
```

**In short:** Your WebSocket connection would be functional but useless — a pipe with no one listening on the other end.
