## Why You Can't Send Objects Directly

WebSocket's `send()` method only accepts specific types:

- `string`
- `Blob`
- `ArrayBuffer`
- `ArrayBufferView`

If you try to send a plain object directly, JavaScript will implicitly call `.toString()` on it, which gives you the useless string `[object Object]` instead of your data:

```javascript
// What you might try
socket.send({ type: 'message', text: 'hello' });

// What the server actually receives
"[object Object]"  // completely useless
```

## What JSON.stringify Does

`JSON.stringify` serializes your object into a JSON string that actually preserves the structure:

```javascript
socket.send(JSON.stringify({ type: 'message', text: 'hello' }));

// What the server actually receives
'{"type":"message","text":"hello"}'  // the server can parse this back
```

## The Round Trip

This is why your server and client both use matching serialize/deserialize calls:

```javascript
// Client sending
socket.send(JSON.stringify({ type: 'message', text: text }));

// Server receiving
const data = JSON.parse(event.data);  // converts string back to object

// Server sending
ws.send(JSON.stringify({ type: 'join', name: name }));

// Client receiving
const data = JSON.parse(event.data);  // converts string back to object
```

The key point is that WebSocket is a **raw bytes transport** and has no built-in concept of objects or types, so you are responsible for serialization yourself. JSON just happens to be the most common convention for text-based WebSocket communication.
