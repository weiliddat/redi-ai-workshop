## Adding Timestamps to Each Message

Your code already receives timestamps from the server and formats them — the issue is that some message types might not show them clearly. Here's a full walkthrough of every step.

---

### Step 1: Understand What You Already Have

In your `onmessage` handler, you already extract the time:

```javascript
const time = new Date(data.timestamp || Date.now()).toLocaleTimeString();
```

This line:
- Reads `data.timestamp` from the server message
- Falls back to `Date.now()` if it's missing
- Converts it to a readable local time like `"2:30:00 PM"`

You already use `time` in your `message`, `join`, and `leave` branches. So timestamps are already working for those three types.

---

### Step 2: Add the Missing Message Types

Your client now needs to handle `nick` and `list` from the extended API. Neither of these currently shows timestamps. Add them inside your `try` block:

```javascript
socket.onmessage = (event) => {
  try {
    const data = JSON.parse(event.data);

    // This line already handles missing timestamps safely
    const time = new Date(data.timestamp || Date.now()).toLocaleTimeString();

    if (data.type === 'message') {
      addMessage(`[${time}] ${data.name}: ${data.text}`);
    } else if (data.type === 'join') {
      addMessage(`[${time}] ${data.name} joined`);
    } else if (data.type === 'leave') {
      addMessage(`[${time}] ${data.name} left`);
    } else if (data.type === 'nick') {
      // Show timestamp alongside the name change notice
      addMessage(`[${time}] ${data.oldName} is now known as ${data.name}`);
    } else if (data.type === 'list') {
      // Join the names array into a readable string
      const nameList = data.names.join(', ');
      addMessage(`[${time}] Online: ${nameList}`);
    } else if (data.type === 'error') {
      // Errors don't have timestamps in the spec, so Date.now() fallback is used
      addMessage(`[ERROR] ${data.message}`);
    }
  } catch (e) {
    addMessage('Invalid message received');
  }
};
```

---

### Step 3: Handle the `/nick` and `/list` Commands in the Form

Your form submit handler currently only sends `message` type. Add command detection before the default send:

```javascript
chatForm.addEventListener('submit', (e) => {
  e.preventDefault();

  if (!socket || socket.readyState !== WebSocket.OPEN) {
    return;
  }

  const text = messageInput.value.trim();
  if (!text) return;

  // Check if the input is a command
  if (text.startsWith('/nick ')) {
    const newName = text.slice(6).trim();   // Remove "/nick " prefix
    if (newName) {
      socket.send(JSON.stringify({ type: 'nick', name: newName }));
    }
  } else if (text === '/list') {
    socket.send(JSON.stringify({ type: 'list' }));
  } else {
    // Regular chat message
    socket.send(JSON.stringify({ type: 'message', text: text }));
  }

  messageInput.value = '';
});
```

---

### Step 4: Optionally Format the Timestamp More Precisely

`toLocaleTimeString()` uses the user's system locale. If you want consistent formatting with hours, minutes, and seconds explicitly:

```javascript
// Replace this:
const time = new Date(data.timestamp || Date.now()).toLocaleTimeString();

// With this:
const time = new Date(data.timestamp || Date.now()).toLocaleTimeString([], {
  hour: '2-digit',
  minute: '2-digit',
  second: '2-digit',
  hour12: true          // Set to false for 24-hour format
});
```

### What Each Message Will Now Look Like

| Server sends | Displayed as |
|---|---|
| `message` | `[2:30:00 PM] Alice: hello` |
| `join` | `[2:30:01 PM] Bob joined` |
| `leave` | `[2:30:05 PM] Bob left` |
| `nick` | `[2:30:10 PM] Alice is now known as AliceNew` |
| `list` | `[2:30:15 PM] Online: Alice, Bob, Charlie` |
| `error` | `[ERROR] name already taken` |

The timestamp always comes from the server's `timestamp` field so all users see the **same time** for the same event, regardless of their local clock.
