This error usually happens in your code **at the moment it tries to process the incoming message**, not necessarily because the message itself is "bad," but because your code tries to read it in a way that JavaScript doesn't understand.

Here are the three most likely scenarios causing `Uncaught SyntaxError: Unexpected token` when a message arrives:

### 1. Malformed JSON (Most Common)
If your message is supposed to be a JSON object (common with WebSockets, API calls, or AJAX), but it is missing a comma, a quote, or a curly brace, parsing it will fail.

**The Error Message:**
If your JSON is missing a closing brace, you might see `Unexpected token }` or `Unexpected token ,` depending on where the parser got stuck.

**The Problem:**
```javascript
// Client-side code (e.g., inside a WebSocket onmessage handler)
const data = JSON.parse(message); // <--- Fails here
```

**The Bad Message:**
```json
{
  "user": "Bob",
  "location": "NY"
}  // <--- Missing a comma after "NY" and a closing "}"
```

**The Fix:**
1.  Open the **Network Tab** in your Developer Tools.
2.  Look for the specific message event.
3.  Click it to view the "Payload" or "Response" (it will look like text).
4.  Copy that text, paste it into a JSON validator (like JSONLint.com). It will tell you exactly what is missing.

---

### 2. HTML or Script Injection
If your application is taking the message and injecting it into the page (e.g., `innerHTML`, `outerHTML`, or assigning text to a `span`) and the message contains a `<script>` tag that has a syntax error inside it.

**The Problem:**
```javascript
// Frontend code
element.innerHTML = message; 

// Incoming Message:
"<script>alert('Missing closing quote</script>"
```
When the browser evaluates this as HTML, it sees the `<script>` tag and tries to run that JavaScript. Since that snippet has a syntax error (missing quote), the JavaScript engine throws a syntax error immediately.

**The Fix:**
*   Ensure you are parsing the data (using `JSON.parse`) **before** putting it into the DOM.
*   Do not inject raw messages into the inner HTML unless you are 100% sure it is safe HTML.

---

### 3. You are using `eval()` on the Message
If your code is dynamic and executes the message content as actual JavaScript code (e.g., receiving "x = 5" and running it), a simple typo in the message will cause this crash.

**The Problem:**
```javascript
// Handling a message that contains code
eval(message);
```

**The Bad Message:**
```javascript
alert("Hello World   // <--- Missing the closing parenthesis
```

**The Fix:**
Avoid `eval()` entirely. If you need to execute code received from a server, ensure the server only sends **data**, not **code**.

### Summary Checklist to Debug
1.  **Where in the code is it failing?** Usually inside your `onmessage` or `.then` handler.
2.  **What is the raw message?** Log it to the console *before* you try to parse it:
    ```javascript
    console.log("Raw message received:", message);
    ```
3.  **Does the message contain special characters?** Check for unclosed quotes or brackets.
4.  **Is it valid JSON?** If the message starts with curly braces `{` but ends differently, or if a string is missing a quote, it won't parse.
