Based on the code you provided, there are a few potential reasons why your message appears twice. However, looking strictly at the **client-side JavaScript**, the most likely culprit is that your **WebSocket Server** is sending the message back (echoing it) twice, or the `onmessage` event is firing twice.

Here is a breakdown of why this is happening and how to fix it.

### The Reason
In the provided code, the client **does not display** the message locally when you click "Send". It only sends the data to the socket and waits for the server to reply.

**1. The Flow:**
1.  You click "Send" (or hit Enter).
2.  `chatForm`'s submit handler sends data to the server: `socket.send(...)`.
3.  The code **does not** call `addMessage()` in this step.
4.  The server receives the message, processes it, and sends it back to *all* connected clients (including you).
5.  The client receives it: `socket.onmessage` triggers.
6.  `addMessage()` creates a `<div>` and appends it to the screen.

**2. Why "Twice"?**
If you see the message twice, it is happening in step 5 or 6. Specifically:
*   **Server Issue:** Your WebSocket server is likely sending the message twice (e.g., because it echoes it back for the sender, and then processes it again, or broadcasts it to two different rooms). This causes `socket.onmessage` to fire twice, appending two divs.
*   **Event Handling:** Rarely, browser quirks can cause the form submit event to behave unexpectedly, but `e.preventDefault()` usually prevents this.

### How to Fix It

You generally have two options to fix this: fix the **Server** (to send it once) or fix the **Client** (to render it only once).

#### Option A: Fix the Client (Recommended for UX)
If you want to see the message appear immediately when you click send (so you know it was sent), you should render it in the `submit` handler. However, to stop it from appearing a second time when the server replies, you need to handle that carefully.

**Here is how to modify your `chatForm` listener:**

```javascript
chatForm.addEventListener('submit', (e) => {
  e.preventDefault();

  if (!socket || socket.readyState !== WebSocket.OPEN) {
    return;
  }

  const text = messageInput.value.trim();
  if (!text) return;

  // 1. Create and display the message immediately (UX)
  const time = new Date().toLocaleTimeString();
  addMessage(`[${time}] You: ${text}`);

  // 2. Send to server
  socket.send(JSON.stringify({
    type: 'message',
    text: text
  }));

  messageInput.value = '';
});
```

*With this change, you will only see one message entry in the chat (from yourself), and when the server replies, it will appear as a separate message from the other user.*

#### Option B: Verify the Server
If you do *not* want the client to render messages immediately, you must ensure your **Server Code** is sending the data only once.
1.  Check your server logs. Are you seeing the message echoed multiple times?
2.  Add a `console.log` inside your `socket.onmessage` handler to check if the event fires twice:

```javascript

    console.log("Message received:", event.data); // Check if this logs twice

```

The code you provided is sending the message correctly. It is receiving the message from the server. If you see duplicates, the server is sending the message back twice. Adding the logic in **Option A** ensures you don't render the duplicate server echo.
