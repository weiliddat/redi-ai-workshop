Your client only renders messages when they come from the server (`socket.onmessage`). It does not locally echo what you send.

So if you see duplicates, it’s almost certainly because the server is sending the same message twice.

The two most common reasons:

1) You’re broadcasting twice on the server  
Example mistake:
- handling the incoming message
- then both echoing back to sender AND broadcasting to all (including sender)

If your broadcast loop already includes the sender, and you also send directly to the sender, they’ll see it twice.

2) You have two active connections  
Less likely here (since you disable the button), but if the server isn’t cleaning up closed sockets properly, it might still think your old connection is alive and send to both.

Quick way to confirm:
- Open dev tools → Network → WS → look at incoming frames  
If you see duplicate frames from the server, it’s 100% server-side.

Fix (typical):
Make sure your server does only one of these:
- broadcast to everyone (including sender), OR
- broadcast to others + send once to sender

Not both.
