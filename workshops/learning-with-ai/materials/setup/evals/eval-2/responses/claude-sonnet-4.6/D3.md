## Uncaught SyntaxError: Unexpected Token

This error typically occurs when you're trying to **parse invalid JSON**. Here's what's likely happening:

### Common Cause

```javascript
// You're probably doing something like this:
socket.on('message', (data) => {
    const parsed = JSON.parse(data); // 💥 Fails if data isn't valid JSON
});
```

---

### Why It Happens

| Scenario | Example of Bad Data |
|----------|-------------------|
| Server sends plain text | `"Hello World"` |
| Server sends HTML error page | `<!DOCTYPE html>...` |
| Malformed JSON | `{name: "John"}` (missing quotes) |
| Undefined/empty value | `undefined` or `` |
| Already-parsed object | Parsing an object twice |

---

### How to Debug It

```javascript
socket.on('message', (data) => {
    // 1. Log raw data BEFORE parsing
    console.log('Raw data:', data);
    console.log('Type:', typeof data);

    try {
        const parsed = JSON.parse(data);
        console.log('Parsed:', parsed);
    } catch (e) {
        console.error('Failed to parse:', e.message);
        console.error('Problematic data:', data);
    }
});
```

---

### Common Fixes

**1. Check if data is already an object**
```javascript
const parsed = typeof data === 'string' ? JSON.parse(data) : data;
```

**2. Validate before parsing**
```javascript
try {
    const parsed = JSON.parse(data);
} catch (e) {
    console.error('Invalid JSON received:', data);
    return; // Handle gracefully
}
```

**3. Fix the server to send valid JSON**
```javascript
// Server side - make sure you're serializing properly
ws.send(JSON.stringify({ message: 'Hello' })); // ✅ Correct
ws.send({ message: 'Hello' });                 // ❌ Wrong
```

---

### Quick Checklist

- [ ] Is the server sending `JSON.stringify()`'d data?
- [ ] Is the client parsing data that's already an object?
- [ ] Is the server returning an HTML error page instead of JSON?
- [ ] Does the JSON have any syntax errors (trailing commas, unquoted keys)?

**What does `console.log('Raw data:', data)` show?** That will pinpoint the exact issue.
