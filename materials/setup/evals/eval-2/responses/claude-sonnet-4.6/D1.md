## WebSocket Connection Failed

This error means **your browser tried to establish a WebSocket connection but couldn't**. Here's what's happening and how to fix it:

---

## Common Causes & Fixes

### 1. **Server isn't running**
The most common cause - nothing is listening on port 3000.
```bash
# Check if something is running on port 3000
lsof -i :3000        # Mac/Linux
netstat -ano | findstr :3000   # Windows
```

### 2. **Server doesn't have WebSocket support**
Your HTTP server needs a WebSocket handler. Example with Node.js:
```javascript
const WebSocket = require('ws');
const wss = new WebSocket.Server({ port: 3000 });

wss.on('connection', (ws) => {
  console.log('Client connected');
  ws.on('message', (msg) => console.log(msg));
});
```

### 3. **Wrong port or URL**
```javascript
// Double-check your client connection URL
const ws = new WebSocket('ws://localhost:3000');  // correct port?
```

### 4. **Proxy blocking the connection**
If using a dev server (Vite, webpack, etc.), configure WebSocket proxying:
```javascript
// vite.config.js
server: {
  proxy: {
    '/ws': {
      target: 'ws://localhost:3000',
      ws: true,          // <-- this is required
    }
  }
}
```

### 5. **HTTPS/WSS mismatch**
```javascript
// If your page is served over HTTPS, you MUST use wss://
const ws = new WebSocket('wss://localhost:3000');  // not ws://
```

## Quick Debugging Steps

| Step | What to check |
|------|--------------|
| Is the server running? | Check terminal/process manager |
| Correct port? | Match server and client port numbers |
| Network tab | Browser DevTools → Network → WS filter |
| Server logs | Look for connection errors on server side |

---

## Key Distinction
- `ws://` = unencrypted (like HTTP)
- `wss://` = encrypted (like HTTPS)
