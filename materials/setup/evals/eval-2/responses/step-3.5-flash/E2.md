If you removed the `socket.onmessage` handler, **the entire chat reception functionality would break**. Here's what konkret would fail:

## **What Would Break:**
1. **No incoming messages displayed** - Users wouldn't see any messages from others (including their own echoed messages)
2. **No join/leave notifications** - Users wouldn't see when people join or leave the chat
3. **No server error messages** - Errors from the server wouldn't be shown to users
4. **No message timestamps** - The timestamp formatting logic (using `data.timestamp`) would never execute

## **What Would Still Work:**
- ✅ **Sending messages** would still work (the `socket.send()` in form submit)
- ✅ **Connection status** would still update (via `onopen`/`onclose`)
- ✅ **Manual disconnect** would still work
- ✅ **URL/name validation** would still work

## **Critical Impact:**
The chat would become **effectively one-way**:
- Users could send messages (server would receive them)
- But users would see **zero feedback** - their own messages wouldn't appear, and they'd see no responses from others
- The chat window would remain static after connection, making it appear broken

## **Why This Happens:**
The `onmessage` handler is the **only place** where:
- `addMessage()` is called to update the UI
- Incoming JSON data is parsed and routed based on `data.type`
- Server responses (including message broadcasts) are processed

Without it, WebSocket messages arrive at the browser but are **silently discarded** since there's no handler to process `event.data`.

## **Minimal Fix Alternative:**
If you wanted the absolute minimum to keep basic chat working, you'd need at least:
```javascript
socket.onmessage = (event) => {
  const data = JSON.parse(event.data);
  if (data.type === 'message') {
    addMessage(`${data.name}: ${data.text}`);
  }
};
```
But you'd still lose timestamps, join/leave notifications, and error handling.

**Bottom line:** Removing `onmessage` makes the chat client a "send-only" device with no visibility into what's happening - essentially breaking the chat functionality from a user's perspective.
