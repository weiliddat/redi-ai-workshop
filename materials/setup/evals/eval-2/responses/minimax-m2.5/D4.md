The issue is almost certainly on your **server** (the backend), not in the HTML/JS you provided.

Here is the breakdown of why this happens:

1.  **Your Client Logic:** Your client code correctly sends a message using `socket.send()` and **does not** manually add the message to the screen (`addMessage`) when you click send. It only adds a message to the window when it receives data from the server inside `socket.onmessage`.
2.  **The Flow:** Every time a message appears in your chat window, it is because the server sent a WebSocket message to your client.
3.  **The Double Send:** If you see the message appearing **twice**, it means your server is sending that specific message to your client **two times** for every one message you send.

### The Likely Cause (Server-Side)

Most chat server implementations make this mistake: they implement both an **Echo** (sending the message specifically back to the sender to confirm receipt) and a **Broadcast** (sending the message to everyone), but they forget that the Broadcast loop also includes the sender.

It likely looks like this on your server (in pseudo-code):

```javascript
ws.on('message', function(data) {
  // 1. You probably have an "Echo" to send it back to the sender specifically
  ws.send(data); 

  // 2. AND a "Broadcast" to everyone (including the sender again!)
  wss.clients.forEach(function(client) {
    client.send(data);
  });
});
```

Because the server does both, your client receives the message twice, triggers `socket.onmessage` twice, and `addMessage` runs twice.

### How to Fix It

You need to modify your server code to do **one or the other**:

1.  **Remove the specific "Echo" to the sender.** If you broadcast to everyone (including the sender), the sender will get the message back naturally. You don't need to `ws.send(data)` specifically to the sender *before* the broadcast.
2.  **Or check the connection.** If you want to keep the Echo, change your Broadcast loop to skip the sender (check `if (client !== ws)`).

Since I cannot see your server code, check your backend logic for where it sends messages out. Look for a place where it iterates over all clients (`forEach`) AND a place where it sends directly to the sender. Remove one of them.
