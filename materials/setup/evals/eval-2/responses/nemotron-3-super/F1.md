To display a timestamp next to each message showing **who sent the message** (not who received it), you need to make one specific change in the client-side JavaScript. Here's exactly what to fix and why:

### The Problem
In the current code, regular messages display the **recipient's name** (e.g., "Bob") instead of the **sender's name** (e.g., "Alice"), because:
- The server sends messages with:
  ```json
  {"type": "message", "oldName": "Alice", "name": "Bob", "text": "Hi", "timestamp": "..."}
  ```
  - `oldName` = sender (Alice)
  - `name` = intended recipient (Bob)
- But the client incorrectly uses `${data.name}` (recipient/Bob) in the display:
  ```javascript
  addMessage(`[${time}] ${data.name}: ${data.text}`); // Shows "[time] Bob: Hi"
  ```

### The Fix
Change **one line** in the `socket.onmessage` handler's `'message'` case:
```diff
- addMessage(`[${time}] ${data.name}: ${data.text}`);
+ addMessage(`[${time}] ${data.oldName}: ${data.text}`);
```

### Why This Works
- After the change, it will use `${data.oldName}` (which contains the **sender's name**, e.g., "Alice")
- The timestamp logic remains correct:
  ```javascript
  const time = new Date(data.timestamp || Date.now()).toLocaleTimeString();
  ```
  - Uses `data.timestamp` when provided by the server (per API spec)
  - Falls back to current time only if timestamp is missing (unlikely per spec)
  - Formats as local time (e.g., "2:35:00 PM") for readability

### Resulting Display
With the API example message:
- **Before fix**: `[14:35:00] Bob: Hi` (shows recipient, confusing)
- **After fix**: `[14:35:00] Alice: Hi` (shows sender + timestamp as intended)

### Verification Against API Spec
- The spec confirms servers **always** include `"timestamp"` for message types
- The spec shows `"oldName"` as the sender's field in server→client broadcasts
- This fix aligns the client's display with the server's naming convention

### Other Message Types Unaffected
- `/list` responses: Already show correct user names (no `oldName`/`name` confusion)
- `join`/`leave` types: Correctly use `${data.name}` (joiner/leaver's name) - no change needed
- Error handling: Unchanged (still shows "Invalid message received")

This single-character-change fix (replacing `data.name` with `data.oldName` in one line) will make timestamps appear next to the **sender's name** for all regular chat messages, matching typical chat app expectations. The timestamp formatting itself was already correct and requires no adjustment.
