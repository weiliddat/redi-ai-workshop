# Why `JSON.stringify` is Needed

The WebSocket `send()` method **cannot send JavaScript objects directly**. It only accepts:
- **Strings** (text)
- **ArrayBuffer**
- **Blob** (binary)

## What Happens If You Don't Stringify?

If you try to send an object directly:

```javascript
socket.send({
  type: 'message',
  text: 'Hello'
});
```

JavaScript automatically converts it to a string using `.toString()`, which produces:

```
"[object Object]"
```

This is useless! You lose all your data.

## The Solution: JSON.stringify

```javascript
// Send as JSON string
socket.send(JSON.stringify({
  type: 'message',
  text: 'Hello'
}));
```

This produces a proper JSON string:

```json
{"type":"message","text":"Hello"}
```

The server can then parse it back:

```javascript
// Server-side (Node.js example)
socket.on('message', (data) => {
  const message = JSON.parse(data);  // Convert string back to object
  console.log(message.type);   // "message"
  console.log(message.text);   // "Hello"
});
```

## Summary

| Step | What Happens |
|------|--------------|
| Send | Object → `JSON.stringify()` → String → WebSocket |
| Receive | String → `JSON.parse()` → Object |

This is the standard pattern for sending structured data over WebSockets.
