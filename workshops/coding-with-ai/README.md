# Coding with AI

This workshop is for people using AI to code on projects.

Participants use one project during the workshop.

This can be a small repo built for the workshop or a pinned commit from an open-source project. Using one project keeps everyone on the same task, setup, and debugging path.

Project work is different from small class exercises:

- the code may affect real users, clients, teammates, or your portfolio
- other people may need to understand your changes
- projects last longer than one session
- unclear changes cost time later

The goal is not just to use AI to finish tasks faster.

The goal is to learn a professional way to code with AI.

## Core Ideas

1. **Use version control liberally**  
   Work in small changes. Commit often. Keep each change focused on one topic.

2. **Build your own understanding**  
   AI can explain, suggest, and draft. You still need to understand the code you accept.

3. **Build the loop as you go**  
   Add tests, run the app, use the browser, take screenshots, and ask for reviews. If you get stuck, update the project instructions so the same problem is easier next time.
