The duplicate lines you see are caused by **the same message being shown twice**:

1. **When you click “Send”** you (or the code you intend to add) immediately display the message in the chat window (a *local echo*).  
2. **The WebSocket server then broadcasts the message to every connected client**, including the one that sent it. Your client receives that broadcast in `socket.onmessage` and calls `addMessage()` a second time.

Because you only have one `addMessage()` call (inside `onmessage`) in the snippet you posted, the most common reason for the double‑appearance is that you **already have a local‑echo line somewhere else in your real code** (for example, you might have added `addMessage(`[${time}] ${name}: ${text}`);` right after `socket.send(...)` in an earlier version, or you are using a library that automatically echoes sent messages). When that line is present, each sent message ends up in the UI twice:

* once from the local‑echo you added yourself, and  
* once from the server’s broadcast that arrives via `onmessage`.

---

### How to verify

Open the browser dev‑tools → **Console** and add a temporary log inside `onmessage`:

```javascript
socket.onmessage = (event) => {
  console.log('RAW WS DATA:', event.data);   // <-- see how many times it fires
  // ... existing parsing ...
};
```

If you see the log printed **twice** for a single send, the server is indeed sending two copies (or you have two listeners). If you see it only once, then the duplicate is coming from a local‑echo you added elsewhere.

---

### Fixes (pick the one that fits your architecture)

#### 1. Remove the local‑echo (recommended if the server already broadcasts to everyone)

```javascript
chatForm.addEventListener('submit', (e) => {
  e.preventDefault();

  if (!socket || socket.readyState !== WebSocket.OPEN) return;

  const text = messageInput.value.trim();
  if (!text) return;

  // ONLY send – do NOT addMessage here
  socket.send(JSON.stringify({ type: 'message', text }));
  messageInput.value = '';
});
```

Now the message appears only when the server’s broadcast arrives in `onmessage`.

#### 2. Keep the local‑echo but tell the client to ignore its own broadcast

If you *do* want instant local feedback (so the user sees their message without waiting for a round‑trip) and you also want the server to rebroadcast to everyone, add a small guard in `onmessage` to drop the echo that originated from you:

```javascript
const myName = nameInput.value.trim();   // capture when you connect

socket.onmessage = (event) => {
  try {
    const data = JSON.parse(event.data);
    const time = new Date(data.timestamp || Date.now()).toLocaleTimeString();

    // Ignore the message we just sent ourselves
    if (data.type === 'message' && data.name === myName) {
      return;   // <-- suppress duplicate
    }

    if (data.type === 'message') {
      addMessage(`[${time}] ${data.name}: ${data.text}`);
    } else if (data.type === 'join') {
      addMessage(`[${time}] ${data.name} joined`);
    } else if (data.type === 'leave') {
      addMessage(`[${time}] ${data.name} left`);
    } else if (data.type === 'error') {
      addMessage(`[ERROR] ${data.message}`);
    }
  } catch (e) {
    addMessage('Invalid message received');
  }
};
```

#### 3. Change the server so it **doesn’t send the message back to the sender**

If you control the WebSocket server, modify the broadcast loop to skip the client that originated the message. (Pseudo‑code for a Node.js `ws` server:)

```javascript
wss.on('connection', (ws, req) => {
  // ... handle join/leave ...

  ws.on('message', (msg) => {
    const data = JSON.parse(msg);
    // broadcast to everyone *except* the sender
    wss.clients.forEach((client) => {
      if (client !== ws && client.readyState === WebSocket.OPEN) {
        client.send(JSON.stringify(data));
      }
    });
  });
});
```

With this change, you can keep the local‑echo (or remove it) and you’ll only see each message once.

---

### TL;DR

*You’re seeing each message twice because you’re showing it locally **and** then again when the server echoes it back to you.*  
Either stop the local echo, filter out your own messages in `onmessage`, or adjust the server so it doesn’t resend the message to the sender. After applying one of those fixes, the chat will display each sent message only once.
