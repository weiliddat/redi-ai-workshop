## ✅ **Complete Updated `onmessage` Handler**

```javascript
socket.onmessage = (event) => {
  try {
    const data = JSON.parse(event.data);
    const time = new Date(data.timestamp || Date.now()).toLocaleTimeString();

    if (data.type === 'message') {
      addMessage(`[${time}] ${data.name}: ${data.text}`);
    } else if (data.type === 'join') {
      addMessage(`[${time}] ${data.name} joined`);
    } else if (data.type === 'leave') {
      addMessage(`[${time}] ${data.name} left`);
    } else if (data.type === 'error') {
      addMessage(`[${time}] [ERROR] ${data.message}`);
    } else if (data.type === 'nick') {
      addMessage(`[${time}] ${data.oldName} changed name to ${data.name}`);
    } else if (data.type === 'list') {
      addMessage(`[${time}] Connected users: ${data.names.join(', ')}`);
    } else {
      console.log('Unknown message type:', data.type);
    }
  } catch (e) {
    const time = new Date().toLocaleTimeString(); // Fallback for parsing errors
    addMessage(`[${time}] Invalid message received`);
  }
};
```

---

## 🔧 **Why This Works**

1. **All server messages include `timestamp`** (ISO 8601 format like `"2025-06-15T14:30:00Z"`).
2. `new Date(data.timestamp)` parses this ISO string correctly.
3. `|| Date.now()` provides fallback if server omits timestamp (unlikely but safe).
4. `toLocaleTimeString()` formats it as `"2:30:45 PM"` (user's local time format).
