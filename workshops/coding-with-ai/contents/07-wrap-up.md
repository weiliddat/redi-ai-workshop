# 07 - Wrap-up

## Goal

End with a simple checklist participants can use in project work.

## The Checklist

Before asking AI to code:

- What is the smallest useful change?
- What files or behavior are relevant?
- What should stay unchanged?

Before accepting AI code:

- Can I explain the diff?
- Did I run the right check?
- What might break?
- Is this change small enough to review?

Before finishing:

- Did I commit the change?
- Does the commit message explain why?
- Did I record any lesson that would help next time?

## Final Message

AI is useful when it helps you ship better work and become a stronger developer.

Use it to move faster, but keep the loop:

1. small change
2. clear understanding
3. strong feedback
4. useful commit
5. better project instructions
