## 1. Add `/nick` Command Detection in Form Submit Handler

Modify the `chatForm.addEventListener('submit')` section to detect and handle `/nick` commands:

```javascript
chatForm.addEventListener('submit', (e) => {
  e.preventDefault();

  if (!socket || socket.readyState !== WebSocket.OPEN) {
    return;
  }

  const text = messageInput.value.trim();
  if (!text) return;

  // Check for /nick command
  if (text.startsWith('/nick ')) {
    const newName = text.substring(6).trim(); // Remove "/nick " prefix
    
    if (!newName) {
      addMessage('[ERROR] Please provide a new name: /nick <new-name>');
      messageInput.value = '';
      return;
    }
    
    // Send nick change request
    socket.send(JSON.stringify({
      type: 'nick',
      name: newName
    }));
    
    messageInput.value = '';
    return; // Don't send as regular message
  }

  // Check for /list command (optional but good to add)
  if (text === '/list') {
    socket.send(JSON.stringify({ type: 'list' }));
    messageInput.value = '';
    return;
  }

  // Regular message
  socket.send(JSON.stringify({
    type: 'message',
    text: text
  }));

  messageInput.value = '';
});
```

## 2. Update Server Response Handling in `onmessage`

Add handling for the new message types (`nick` and `list`) in your `socket.onmessage`:

```javascript
socket.onmessage = (event) => {
  try {
    const data = JSON.parse(event.data);
    const time = new Date(data.timestamp || Date.now()).toLocaleTimeString();

    switch (data.type) {
      case 'message':
        addMessage(`[${time}] ${data.name}: ${data.text}`);
        break;
        
      case 'join':
        addMessage(`[${time}] ${data.name} joined`);
        break;
        
      case 'leave':
        addMessage(`[${time}] ${data.name} left`);
        break;
        
      case 'nick':
        // Server broadcasts nick change to everyone
        addMessage(`[${time}] ${data.oldName} is now known as ${data.name}`);
        break;
        
      case 'list':
        // Server sends user list only to requester
        const userList = data.names.join(', ');
        addMessage(`[${time}] Connected users: ${userList}`);
        break;
        
      case 'error':
        addMessage(`[ERROR] ${data.message}`);
        break;
        
      default:
        addMessage(`[${time}] Unknown message type: ${data.type}`);
    }
  } catch (e) {
    addMessage('Invalid message received');
  }
};
```

## What Happens End-to-End:

1. **User types**: `/nick Bob`
2. **Client sends**: `{"type":"nick","name":"Bob"}`
3. **Server validates**:
   - If `"Bob"` is available → broadcasts: `{"type":"nick","oldName":"Alice","name":"Bob","timestamp":"..."}` to **all** clients
   - If `"Bob"` is taken → sends only to you: `{"type":"error","message":"name already taken"}`
4. **Client receives**:
   - Success: Updates `currentName`, changes input field, shows broadcast message
   - Error: Shows error message

## Important Notes:

- The **server broadcasts** `nick` messages to **all** clients, not just the requester
- The **server sends** `list` responses **only to the requester**
- Your `currentName` variable should be the **single source of truth** for "who you are" in the current session
- Consider **disabling** the `nameInput` field while connected (as shown above) to prevent confusion
- The server may have **additional validation** (e.g., name length, allowed characters) that could return errors
