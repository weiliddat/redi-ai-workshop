The reason your message appears twice is likely due to **how your WebSocket server is broadcasting messages**.

In most chat server implementations, when a client sends a message, the server receives it and then broadcasts it back to **all connected clients**, including the sender.

Here is the breakdown of what is happening:

1.  **Client A (You) sends a message.**
2.  **Client A adds the message to the UI.** (Your client-side code adds it immediately when you click Send).
3.  **Server receives the message** and broadcasts it to everyone (including Client A).
4.  **Client A receives the broadcast** and adds it to the UI a second time.

Since your client-side code does not have any logic to prevent adding messages from yourself, you see it twice.

---

### How to Fix It

You need to modify the client to distinguish between messages you sent and messages received from others. You can do this by having the server include a `senderId` or `clientId` in the message data, and then ignoring messages on the client where `senderId` matches your own.

Here is the updated client-side JavaScript logic to handle this:

#### 1. Generate a Unique ID for the Client
When the script loads, generate a unique ID for this browser instance.

#### 2. Update the `onmessage` Logic
Check if the incoming message belongs to you. If it does, ignore it (because you already added it locally).

Here is the modified script section:

```html
<script>
    let socket = null;
    // 1. Generate a unique ID for this client session
    const myClientId = 'client_' + Math.random().toString(36).substr(2, 9);

    const serverUrlInput = document.getElementById('serverUrl');
    const nameInput = document.getElementById('nameInput');
    const connectBtn = document.getElementById('connectBtn');
    const disconnectBtn = document.getElementById('disconnectBtn');
    const statusEl = document.getElementById('status');
    const messagesEl = document.getElementById('messages');
    const chatForm = document.getElementById('chatForm');
    const messageInput = document.getElementById('messageInput');

    function setStatus(text) {
      statusEl.textContent = text;
    }

    function addMessage(text) {
      const div = document.createElement('div');
      div.className = 'message';
      div.textContent = text;
      messagesEl.appendChild(div);
      messagesEl.scrollTop = messagesEl.scrollHeight;
    }

    function connect() {
      const baseUrl = serverUrlInput.value.trim();
      const name = nameInput.value.trim();

      if (!baseUrl || !name) {
        alert('Please enter server URL and name');
        return;
      }

      // Pass the client ID to the server via query param (optional but useful)
      const url = baseUrl + '?name=' + encodeURIComponent(name) + '&id=' + myClientId;

      socket = new WebSocket(url);

      socket.onopen = () => {
        setStatus('Connected');
        connectBtn.disabled = true;
        disconnectBtn.disabled = false;
      };

      socket.onclose = () => {
        setStatus('Disconnected');
        connectBtn.disabled = false;
        disconnectBtn.disabled = true;
        socket = null;
      };

      socket.onerror = () => {
        setStatus('Disconnected');
      };

      socket.onmessage = (event) => {
        try {
          const data = JSON.parse(event.data);

          // 2. Check if the message is from THIS client
          // If the server sends back an 'id' field and it matches ours, skip adding it
          if (data.id === myClientId) {
             return; // Exit early, we already displayed this message when we sent it
          }

          const time = new Date(data.timestamp || Date.now()).toLocaleTimeString();

          if (data.type === 'message') {
            addMessage(`[${time}] ${data.name}: ${data.text}`);
          } else if (data.type === 'join') {
            addMessage(`[${time}] ${data.name} joined`);
          } else if (data.type === 'leave') {
            addMessage(`[${time}] ${data.name} left`);
          } else if (data.type === 'error') {
            addMessage(`[ERROR] ${data.message}`);
          }
        } catch (e) {
          addMessage('Invalid message received');
        }
      };
    }

    function disconnect() {
      if (socket) {
        socket.close();
      }
    }

    connectBtn.addEventListener('click', connect);
    disconnectBtn.addEventListener('click', disconnect);

    chatForm.addEventListener('submit', (e) => {
      e.preventDefault();

      if (!socket || socket.readyState !== WebSocket.OPEN) {
        return;
      }

      const text = messageInput.value.trim();
      if (!text) return;

      // 3. Send the message with your Client ID
      socket.send(JSON.stringify({
        type: 'message',
        text: text,
        id: myClientId // Include ID in the payload
      }));

      // OPTIONAL: If you want the message to appear immediately without waiting for the server echo:
      // const time = new Date().toLocaleTimeString();
      // addMessage(`[${time}] ${nameInput.value.trim()}: ${text}`);

      messageInput.value = '';
    });
</script>
```

### Important: You may also need to update your Server

For the client-side check `if (data.id === myClientId)` to work, your **server** must echo the `id` back to the client.

When your server receives a message from a client, it currently probably looks like this:
`{ type: 'message', name: 'John', text: 'Hello' }`

It needs to look like this:
`{ type: 'message', name: 'John', text: 'Hello', id: 'client_12345' }`

**Example Node.js Server Fix:**

If you are using Node.js, your broadcast function should look something like this:

```javascript
// Inside your 'connection' event
ws.on('message', (message) => {
  try {
    const data = JSON.parse(message);
    
    // Add the sender's ID to the object before broadcasting
    const payload = {
      type: 'message',
      name: ws.name, // stored during connection
      text: data.text,
      timestamp: Date.now(),
      id: data.id // Echo the ID sent by the client
    };

    // Broadcast to all clients (including sender)
    wss.clients.forEach((client) => {
      if (client.readyState === WebSocket.OPEN) {
        client.send(JSON.stringify(payload));
      }
    });
  } catch (e) {
    console.error('Error parsing message');
  }
});
```

### Alternative: The "Echo" Logic (No Server Changes)
If you cannot modify the server to send back the `id`, you can use a temporary "pending message" cache, but that is more complex and prone to race conditions.

The **ID check method above is the standard and most robust solution**.
