This error means **your WebSocket server is sending data that isn't valid JSON**, or your client is trying to parse non-JSON data as JSON. Here's what's happening and how to fix it:

## Common Causes & Solutions

### 1. **Server sending plain text instead of JSON**
If your server sends raw text like `"Hello"` instead of JSON `{"message": "Hello"}`:

**Fix server-side:**
```javascript
// WRONG (sends plain text)
ws.send("Hello");

// CORRECT (sends JSON)
ws.send(JSON.stringify({ message: "Hello" }));
```

### 2. **Double stringification or malformed JSON**
If you accidentally stringify twice:
```javascript
// WRONG: Double stringification
ws.send(JSON.stringify(JSON.stringify({ message: "Hello" })));
// Result: "\"{\\\"message\\\":\\\"Hello\\\"}\"" (invalid)

// CORRECT:
ws.send(JSON.stringify({ message: "Hello" }));
```

### 3. **BOM (Byte Order Mark) or invisible characters**
Some editors/files add BOM characters (`\ufeff`) at the start.

**Fix client-side:**
```javascript
function receiveMessage(event) {
  try {
    // Clean the data first
    const cleanData = event.data.replace(/^\uFEFF/, '').trim();
    const messageData = JSON.parse(cleanData);
    
    if (messageData.message) {
      // ... rest of your code
    }
  } catch (error) {
    console.error('Failed to parse:', event.data, error);
  }
}
```

### 4. **Debug what's actually being sent**
Add this to see what you're receiving:

**Client-side debug:**
```javascript
function receiveMessage(event) {
  console.log('Raw data received:', typeof event.data, event.data);
  console.log('First 50 chars:', event.data.substring(0, 50));
  
  try {
    const messageData = JSON.parse(event.data);
    // ... rest
  } catch (error) {
    console.error('Parse failed. Showing as string:', event.data);
  }
}
```

**Server-side debug:**
```javascript
// Node.js example
ws.on('message', (msg) => {
  console.log('Sending to clients:', typeof msg, msg);
  // Should be stringified JSON
});
```

## Quick Diagnostic Checklist

1. **Check server logs** - What data is actually being sent?
2. **Test with a tool** - Use `wscat` or Postman to see raw WebSocket messages:
   ```bash
   wscat -c ws://localhost:8080
   > {"message": "test"}
   ```
3. **Validate JSON** - Copy the received data into [jsonlint.com](https://jsonlint.com)
4. **Check for array vs object** - Are you sending `[{...}]` but expecting `{...}`?
