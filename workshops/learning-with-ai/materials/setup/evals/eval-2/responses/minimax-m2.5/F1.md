Below is a **step‑by‑step** guide that explains what you need to do in order to display a timestamp **next to every message** that comes from the server (including the special “join”, “leave”, “nick”, “list” and “error” messages).  
The explanation is followed by the minimal code changes you can drop into your existing `socket.onmessage` handler.

---

## 1.  What the server already sends you  

According to the API spec, every broadcast packet contains a `timestamp` field:

```json
{
  "type": "message",
  "name": "Alice",
  "text": "Hi there",
  "timestamp": "2025-06-15T14:30:00Z"
}
```

(The same is true for `join`, `leave`, `nick` and `list` – `error` does **not** have a timestamp, but we’ll handle that on the client side.)

So the *server* already does the heavy lifting; you only have to **read** that field and **format** it for the user.

---

## 2.  Parse the incoming JSON  

In your `socket.onmessage` callback you already call `JSON.parse(event.data)`.  
After parsing you have a plain JavaScript object (`data`) that contains the fields described above.

---

## 3.  Extract the timestamp (with a fallback)  

Sometimes the field may be missing (e.g., an error packet). Use a short‑circuit that falls back to the current time:

```js
const ts = data.timestamp ? new Date(data.timestamp) : new Date();
```

* `new Date(...)` works with both an ISO‑8601 string (`"2025‑06‑15T14:30:00Z"`) and a numeric milliseconds value, so it’s safe for any format the server might send.

---

## 4.  Convert the timestamp into a human‑readable string  

The simplest way is to use the built‑in locale‑aware formatter:

```js
const timeStr = ts.toLocaleTimeString();           // e.g. "2:30:00 PM"
```

If you also want the **date** (useful when the chat runs across midnight), use `toLocaleString()` instead:

```js
const dateTimeStr = ts.toLocaleString();           // e.g. "6/15/2025, 2:30:00 PM"
```

Pick the version you prefer – the rest of the code works with either one.

---

## 5.  Build the message text that includes the timestamp  

For each message type, concatenate the timestamp in front of the rest of the text.  
Example for a normal chat message:

```js
addMessage(`[${timeStr}] ${data.name}: ${data.text}`);
```

You can do the same for `join`, `leave`, `nick`, and even for `list` (if you want to show when the list was generated).  

For the **error** packet the server does not send a timestamp, so we use the fallback we created in step 3:

```js
addMessage(`[${timeStr}] [ERROR] ${data.message}`);
```

---

## 6.  Update the UI (the `addMessage` helper)  

Your current `addMessage` simply appends a `<div>` with `textContent`.  
If you want the timestamp to be visually distinct (e.g., lighter colour), you can wrap it in a `<span>`:

```js
function addMessage(html) {
  const div = document.createElement('div');
  div.className = 'message';
  div.innerHTML = html;               // ← we now pass HTML, not plain text
  messagesEl.appendChild(div);
  messagesEl.scrollTop = messagesEl.scrollHeight;
}
```

Then call it like this:

```js
addMessage(`<span class="timestamp">[${timeStr}]</span> ${data.name}: ${data.text}`);
```

Add a tiny CSS rule to style timestamps:

```css
.timestamp {
  color: #888;
  font-size: 0.85em;
  margin-right: 4px;
}
```

---

## 7.  Full, updated `socket.onmessage` handler  

Below is the complete, drop‑in replacement for the existing `socket.onmessage` callback. It covers **all** message types the server can send, adds a timestamp to each of them, and uses a fallback for packets that lack a timestamp.

```javascript
socket.onmessage = (event) => {
  let data;
  try {
    data = JSON.parse(event.data);
  } catch (e) {
    addMessage('Invalid message received');
    return;
  }

  // ---- timestamp -------------------------------------------------
  // Use server timestamp if present, otherwise fallback to now.
  const ts        = data.timestamp ? new Date(data.timestamp) : new Date();
  const timeStr   = ts.toLocaleTimeString();          // “2:30:00 PM”
  // -----------------------------------------------------------------

  switch (data.type) {
    case 'message':
      // Normal chat message
      addMessage(
        `<span class="timestamp">[${timeStr}]</span> ${data.name}: ${data.text}`
      );
      break;

    case 'join':
      // User joined the room
      addMessage(
        `<span class="timestamp">[${timeStr}]</span> ${data.name} joined`
      );
      break;

    case 'leave':
      // User left the room
      addMessage(
        `<span class="timestamp">[${timeStr}]</span> ${data.name} left`
      );
      break;

    case 'nick':
      // Name change notification
      addMessage(
        `<span class="timestamp">[${timeStr}]</span> ${data.oldName} changed name to ${data.name}`
      );
      break;

    case 'list':
      // List of currently connected users
      addMessage(
        `<span class="timestamp">[${timeStr}]</span> Users online: ${data.names.join(', ')}`
      );
      break;

    case 'error':
      // Server‑generated error (no timestamp from server, use our fallback)
      addMessage(
        `<span class="timestamp">[${timeStr}]</span> <strong>ERROR:</strong> ${data.message}`
      );
      break;

    default:
      // Ignore unknown packet types (or log them for debugging)
      // console.warn('Unknown message type:', data.type);
      break;
  }
};
```

---

## 8.  (Optional) Show a timestamp for *your own* messages immediately  

If you want the user to see their own text **right away** – without waiting for the server to echo it back – you can prepend a timestamp at the moment you send the message:

```javascript
chatForm.addEventListener('submit', (e) => {
  e.preventDefault();

  if (!socket || socket.readyState !== WebSocket.OPEN) return;

  const text = messageInput.value.trim();
  if (!text) return;

  // Build the payload exactly as before
  const payload = JSON.stringify({
    type: 'message',
    text: text
  });

  // Optional: show the message locally with a timestamp right now
  const now = new Date().toLocaleTimeString();
  addMessage(`<span class="timestamp">[${now}]</span> <strong>You:</strong> ${text}`);

  socket.send(payload);
  messageInput.value = '';
});
```

> **Why is this optional?**  
> The server will broadcast the same message back to you (including a proper server‑generated timestamp). If you display it twice you’ll end up with duplicates. Most developers let the server echo the message back and simply rely on the code in step 7. The snippet above is useful only if you prefer an *instant* UI update.

---

### Recap of the steps

| Step | What you do |
|------|--------------|
| **1** | Trust the server to put a `timestamp` field in every broadcast (the spec guarantees it). |
| **2** | In `socket.onmessage`, parse the JSON (`JSON.parse`). |
| **3** | Extract `data.timestamp`; if missing, fall back to `Date.now()`. |
| **4** | Convert the timestamp to a readable string (`toLocaleTimeString()` or `toLocaleString()`). |
| **5** | Build the message string **with** that timestamp (and a `<span>` if you want custom styling). |
| **6** | Send the string to your existing `addMessage` helper (or tweak the helper to accept HTML). |
| **7** | (Optional) Add a tiny CSS rule to make timestamps look subtle. |
| **8** | (Optional) Show a timestamp immediately when the user sends a message. |

That’s it! With those changes every line that appears in the chat window will have a timestamp right next to it, and you’ll also see timestamps for join/leave, name‑change, list, and error notifications. Happy coding!
