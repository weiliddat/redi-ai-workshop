# Why `JSON.stringify` is Needed

The short answer: **WebSocket's `send()` method only accepts strings, Blobs, or ArrayBuffers—not plain JavaScript objects**.

## What Happens Internally

```javascript
// What you write:
socket.send({ type: 'message', text: 'Hello' });

// What the browser actually needs:
socket.send('{"type":"message","text":"Hello"}');
```

The `send()` method doesn't know how to serialize a JavaScript object into bytes for transmission. You must convert it to a string first.

## The Full Round Trip

```
Sender (Client)                          Receiver (Server)
     │                                         │
     │  JSON.stringify({...})                  │
     │  ──────────────────────────────────►   │
     │  '{"type":"message","text":"Hi"}'       │
     │                                         │
     │                        JSON.parse(str)  │
     │                        ◄──────────────  │
     │  { type: 'message', text: 'Hi' }        │
```

## Can You Send Objects Directly?

**No**, but here's what you *can* send:

| Type | Example | Works? |
|------|---------|--------|
| Plain object | `{ type: 'message' }` | ❌ No |
| JSON string | `'{"type":"message"}'` | ✅ Yes |
| Plain string | `'Hello world'` | ✅ Yes |
| Blob | `new Blob(['data'])` | ✅ Yes |
| ArrayBuffer | `new ArrayBuffer(8)` | ✅ Yes |

## Summary

- **`JSON.stringify`** = JavaScript object → JSON string (for sending)
- **`JSON.parse`** = JSON string → JavaScript object (for receiving)

This serialization step is how client and server agree on a common "language" for exchanging structured data.
