## The Double Message Problem

Your client code itself is **not** the cause — the issue is almost certainly in your **server code**.

Here's what's happening:

1. You send a message to the server
2. The server broadcasts it to **all connected clients**, including the sender
3. Your client displays it once from the server's broadcast
4. But your server is likely **also echoing it back separately**, causing the duplicate

### What to look for in your server code

The typical mistake looks like this:

```javascript
// Server-side (the bug)
ws.on('message', (data) => {
  ws.send(data);           // ← echoes back to sender (1st display)
  broadcast(data);         // ← broadcast includes sender again (2nd display)
});
```

### The fix

Either **exclude the sender from the broadcast**:

```javascript
ws.on('message', (data) => {
  clients.forEach((client) => {
    if (client !== ws && client.readyState === WebSocket.OPEN) { // ← skip sender
      client.send(data);
    }
  });
});
```

Or **include the sender in the broadcast but remove the separate echo**:

```javascript
ws.on('message', (data) => {
  clients.forEach((client) => {
    if (client.readyState === WebSocket.OPEN) {
      client.send(data); // sender gets it once via broadcast
    }
  });
  // ← no separate ws.send()
});
```

### Why your client code is fine

Your client only displays messages received from the server via `socket.onmessage` — it does **not** optimistically add the message locally when you hit send, so it's entirely dependent on what the server sends back. One echo from the server = one message displayed.
