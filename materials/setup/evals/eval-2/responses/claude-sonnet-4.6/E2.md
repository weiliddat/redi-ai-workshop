## What Would Break Without the `onmessage` Handler

Removing the `onmessage` handler would mean **all incoming server messages are silently ignored**. Here's what specifically breaks:

### Features That Would Stop Working

| Feature | Why It Breaks |
|---|---|
| **Chat messages** | `data.type === 'message'` handler never fires, no messages appear |
| **Join notifications** | `data.type === 'join'` never displayed |
| **Leave notifications** | `data.type === 'leave'` never displayed |
| **Server error messages** | `data.type === 'error'` never shown to user |

### What Would Still Work

- Connecting to the server
- The `onopen`, `onclose`, `onerror` handlers still fire normally
- **Sending** messages still works since that uses `socket.send()`, which is independent
- Status updates ("Connected" / "Disconnected") still display

### The Core Problem

```javascript
// Without this, the WebSocket receives data but has nothing to do with it
socket.onmessage = (event) => {
    // This entire block would be gone
};
```

The connection becomes **one-way** — the client can send messages but would have **no way to display anything received back**, making it effectively a broken chat application since you'd be typing into a void with no visible responses or activity from other users.
