# Hands-On: Own Track

## Setup

Students apply the same learning strategies to a task in their own track. This time, the goal is theirs — something they've been wanting to build or a problem they've been stuck on.

The teacher gives the prompt:

> **"Think of something in your track you've been wanting to build or understand. Use AI to work on it — but use the strategies you just practiced. Ask why. Ask how. Don't just accept the output."**

If students need ideas:

### Frontend (HTML/CSS/React)

- Build a dark mode toggle that saves the preference
- Create a form with client-side validation
- Make an existing page responsive

### Backend (Python)

- Build a simple REST endpoint that reads/writes to a JSON file
- Write a script that fetches data from a public API and filters it
- Add error handling to an existing function and understand each case

### Data Science

- Load a dataset, clean it, and produce a summary chart
- Write a function that detects outliers and explain the method
- Compare two approaches to the same analysis and explain tradeoffs

## Teacher's role

Walk the Zoom breakout rooms (or main room). Look for:

- Students asking AI _why_, not just _what_
- Students who can explain what their code does when you ask
- Students falling back into delegation — gently redirect: _"Can you explain that line to me?"_

## Time

- 20 min — open work (optional section, depending on time/progress)
