# Adding the `/nick` Command

You need to make two changes to your client code:

## 1. Handle the `nick` message type in `socket.onmessage`

Add a new handler for when the server broadcasts a name change (could be your own change or another user's):

```javascript
socket.onmessage = (event) => {
  try {
    const data = JSON.parse(event.data);

    const time = new Date(data.timestamp || Date.now()).toLocaleTimeString();

    if (data.type === 'message') {
      addMessage(`[${time}] ${data.name}: ${data.text}`);
    } else if (data.type === 'join') {
      addMessage(`[${time}] ${data.name} joined`);
    } else if (data.type === 'leave') {
      addMessage(`[${time}] ${data.name} left`);
    } else if (data.type === 'nick') {                           // ADD THIS
      addMessage(`[${time}] ${data.oldName} is now known as ${data.name}`);
    } else if (data.type === 'error') {
      addMessage(`[ERROR] ${data.message}`);
    }
  } catch (e) {
    addMessage('Invalid message received');
  }
};
```

## 2. Detect `/nick` commands in the form submit handler

Modify your `chatForm` submit listener to intercept `/nick` commands:

```javascript
chatForm.addEventListener('submit', (e) => {
  e.preventDefault();

  if (!socket || socket.readyState !== WebSocket.OPEN) {
    return;
  }

  const text = messageInput.value.trim();
  if (!text) return;

  // Check for /nick command
  if (text.toLowerCase().startsWith('/nick ')) {
    const newName = text.substring(6).trim(); // Extract name after "/nick "
    
    if (!newName) {
      alert('Please provide a new name');
      return;
    }

    socket.send(JSON.stringify({
      type: 'nick',
      name: newName
    }));
    
    messageInput.value = '';
    return;
  }

  // Regular message sending
  socket.send(JSON.stringify({
    type: 'message',
    text: text
  }));

  messageInput.value = '';
});
```

## Summary of changes

| Location | Change |
|----------|--------|
| `socket.onmessage` | Added `nick` case to display "X is now known as Y" |
| `chatForm` submit | Added check for `/nick ` prefix, sends `{"type": "nick", "name": "..."}` instead of a regular message |

The client will now:
1. Detect when you type `/nick newname`
2. Send the nick change request to the server
3. Display a message when anyone (including you) changes their name
4. Show errors if the name is already taken (via the existing error handler)
