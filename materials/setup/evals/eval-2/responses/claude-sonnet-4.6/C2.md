## Callback Functions

A **callback function** is a function you pass to something else, to be called **later** when a specific event happens.

Instead of running code *right now*, you're saying:
> *"When this thing happens, run this code."*

---

## Why Not Write the Code Directly?

Consider this example:

```javascript
// You might wonder: why not just do this?
let data = ws.onmessage  // just read the message directly?
```

The problem is **you don't know when a message will arrive.**

WebSocket messages come in **asynchronously** - they could arrive in:
- 50ms
- 5 seconds
- Never

JavaScript can't just *stop and wait* for a message, because that would **freeze the entire page.**

---

## How Callbacks Solve This

```javascript
// You hand the WebSocket a function to call LATER
ws.onmessage = function(event) {
    console.log(event.data)  // runs only when a message arrives
}

// This code runs immediately, without waiting
console.log("This runs right away")
```

The flow is:
1. You register the callback
2. JavaScript continues running other code
3. A message arrives later
4. The WebSocket **calls your function** automatically

---

## A Simple Analogy

It's like leaving your number at a restaurant:

```
You:       "Call me when my table is ready"
           └── this is the callback (your phone number)

Restaurant: *calls you later* when the table is ready
            └── this is the event trigger
```

You don't stand at the door staring - you go do other things and get notified when needed.

---

## Key Point

| Approach | Problem |
|----------|---------|
| Write code directly | Runs *once*, right now, before any message exists |
| Use a callback | Runs *every time* a message arrives, no matter when |

The callback lets the WebSocket say **"I'll call you back"** when something actually happens.
