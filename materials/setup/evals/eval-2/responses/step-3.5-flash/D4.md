Looking at your client code, the most likely reason your message appears twice is **because the server is echoing your message back to you** *in addition to* broadcasting it to other clients. This is a common design pattern where:

1. You send a message to the server
2. The server broadcasts it to ALL connected clients (including you)
3. Your client receives and displays it

But if your server is **also** sending a direct "acknowledgment" or "echo" message specifically back to you (in addition to the broadcast), you'd see it twice.

## Why this happens:

**Typical flow that causes duplication:**
```
You → Server: {type: "message", text: "Hello"}
Server → You: {type: "message", text: "Hello"}  // Direct echo
Server → EveryoneElse: {type: "message", text: "Hello"}  // Broadcast
Server → You: {type: "message", text: "Hello"}  // You're part of "everyone" in broadcast
```

Your client displays both the direct echo AND the broadcast version.

## How to fix this:

### Option 1: Fix the server (Recommended)
The server should **only broadcast once**, not send both a direct echo and a broadcast. Your server code should look something like:

```javascript
// When server receives a message:
socket.on('message', (data) => {
  // Broadcast to ALL clients (including sender)
  io.emit('message', {
    name: socket.name,
    text: data.text,
    timestamp: Date.now()
  });
});
```

**NOT**:
```javascript
// WRONG: Sending twice
socket.emit('message', ...);  // Direct to sender
io.emit('message', ...);     // Broadcast to all (including sender again)
```

### Option 2: Client-side deduplication (Temporary fix)
If you can't modify the server, add deduplication logic:

```javascript
let lastMessage = null;

socket.onmessage = (event) => {
  try {
    const data = JSON.parse(event.data);
    const time = new Date(data.timestamp || Date.now()).toLocaleTimeString();
    const messageText = `[${time}] ${data.name}: ${data.text}`;
    
    // Check if this is a duplicate of the last message
    if (messageText === lastMessage) {
      return; // Skip duplicate
    }
    
    lastMessage = messageText;
    
    if (data.type === 'message') {
      addMessage(messageText);
    } else if (data.type === 'join') {
      addMessage(`[${time}] ${data.name} joined`);
    } else if (data.type === 'leave') {
      addMessage(`[${time}] ${data.name} left`);
    } else if (data.type === 'error') {
      addMessage(`[ERROR] ${data.message}`);
    }
  } catch (e) {
    addMessage('Invalid message received');
  }
};
```

### Option 3: Server should track sender
The cleanest solution is for the server to **not send the message back to the sender** if you only want to see your messages after they've been broadcast:

```javascript
// Server: Broadcast to everyone EXCEPT sender
socket.broadcast.emit('message', {
  name: socket.name,
  text: data.text,
  timestamp: Date.now()
});

// Then separately, if you want the sender to see their own message:
socket.emit('message', {
  name: socket.name,
  text: data.text,
  timestamp: Date.now(),
  echo: true  // Mark it as an echo if needed
});
```

But this is more complex than just broadcasting once to everyone.

## Check your server code:
Look for where you handle incoming messages. You should see only **one** emit/broadcast call per message, not two.

**What you should NOT see:**
```javascript
// BAD: Two sends
socket.emit('message', response);    // 1st send to same client

```

**What you should see:**
```javascript
// GOOD: One broadcast to all
io.emit('message', response);
// OR
socket.broadcast.emit('message', response); // To all EXCEPT sender
```
